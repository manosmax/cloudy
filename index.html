Overview

What Was Built

A fully functional personal cloud storage server — a self-hosted alternative to Google Drive — running on an old i3 prebuilt desktop PC. Accessible from anywhere in the world at https://googlecantdrive.uk with a real SSL certificate and no open router ports.

Multiple user accounts can be created with individual storage quotas. Files sync automatically via official Nextcloud desktop and mobile apps on any platform — Windows, Mac, Linux, iOS and Android.

The entire stack runs in Docker containers, making it trivial to update, back up, and maintain. The server requires no monitor or keyboard — managed entirely over SSH from a laptop.

Build Process

How It Was Done

01
OS Ubuntu Server
▼

Ubuntu Server was installed as the operating system. Unlike the desktop edition it ships with no graphical interface, leaving maximum resources for Docker and Nextcloud. It is purpose-built for always-on headless machines managed entirely over SSH — the ideal foundation for a home server.

02
Infrastructure Docker & Docker Compose
▼

Docker was installed to run all services as isolated containers, eliminating dependency conflicts and making updates and backups straightforward. Three containers were deployed via a single Compose file:

  • MariaDB — relational database storing all user data and file metadata
  • Redis — in-memory cache providing file locking, preventing corruption under concurrent access
  • Nextcloud — the cloud storage application itself
03
Application Nextcloud
▼

Nextcloud is a production-grade open-source Google Drive alternative. It was connected to MariaDB for persistent data storage and Redis for performance. The admin account and database credentials were configured via environment variables in Docker Compose on first launch, avoiding any manual setup wizard interaction.

04
Networking Static IP Address
▼

The server was assigned a permanent local IP address of 192.168.1.89 using Netplan — Ubuntu's network configuration system. Without a static assignment, the router's DHCP server could assign a different IP on each reboot, silently breaking SSH access, Nextcloud's trusted domain allowlist, and the Cloudflare tunnel configuration.

05
Remote Access SSH Server
▼

OpenSSH Server was installed and enabled to start automatically at every boot via systemd. This allows the entire server to be administered from a laptop over the local network — no monitor, keyboard, or physical interaction ever required. The machine needs only power and ethernet to operate indefinitely.

06
Hardening Nextcloud Configuration
▼

Post-install configuration was applied via the occ CLI tool to clear all admin warnings and tune performance:

  • Trusted Domains — restricts access to known addresses only
  • Maintenance Window — heavy background jobs scheduled for 2AM
  • Mimetype Migrations — ensures correct file type handling
  • Database Indexes — missing indexes added for faster queries
  • Phone Region — set to GR for correct number formatting
  • Protocol Override — forced to HTTPS after tunnel activation
07
Internet Exposure Cloudflare Tunnel
▼

A Cloudflare Tunnel exposes the server to the internet without opening any inbound ports on the router. The cloudflared daemon maintains a persistent outbound encrypted connection to Cloudflare's edge network. Cloudflare forwards requests arriving at https://googlecantdrive.uk through that tunnel to Nextcloud on port 8080. Free SSL/TLS termination is included. The daemon is registered as a systemd service, ensuring it restarts automatically after every reboot or crash.
